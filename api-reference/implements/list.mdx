# List Implements

Retrieve all agricultural implements from the catalog with pagination and optional search filters. This is a public endpoint that does not require authentication.

## Endpoint

```
GET /api/implements
```

## Query Parameters

<ParamField query="limit" type="integer" default="10">
  Number of records per page (pagination limit)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of records to skip (pagination offset)
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number (alternative to offset)
</ParamField>

<ParamField query="sort" type="string">
  Field to sort by (e.g., `implement_type`, `brand`, `power_requirement_hp`)
</ParamField>

<ParamField query="order" type="string" default="asc">
  Sort order: `asc` or `desc`
</ParamField>

## Response

Returns a paginated list of implements with metadata.

### Response Schema

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="array">
  Array of implement objects
  
  <Expandable title="Implement Object">
    <ResponseField name="implement_id" type="integer">
      Unique implement identifier
    </ResponseField>
    
    <ResponseField name="implement_name" type="string">
      Name of the implement
    </ResponseField>
    
    <ResponseField name="brand" type="string">
      Manufacturer brand
    </ResponseField>
    
    <ResponseField name="power_requirement_hp" type="number">
      Required horsepower (HP) to operate the implement
    </ResponseField>
    
    <ResponseField name="working_width_m" type="number">
      Working width in meters
    </ResponseField>
    
    <ResponseField name="soil_type" type="string">
      Compatible soil type (e.g., "Loam", "Clay", "Sandy", "All")
    </ResponseField>
    
    <ResponseField name="working_depth_cm" type="number">
      Working depth in centimeters (for plows, cultivators)
    </ResponseField>
    
    <ResponseField name="weight_kg" type="number">
      Implement weight in kilograms
    </ResponseField>
    
    <ResponseField name="implement_type" type="string">
      Type of implement. Valid values:
      - `plow` - Disc or moldboard plows
      - `harrow` - Disc or tine harrows
      - `seeder` - Seed planting equipment
      - `sprayer` - Pesticide/fertilizer sprayers
      - `harvester` - Harvesting equipment
      - `cultivator` - Soil cultivation tools
      - `mower` - Mowing equipment
      - `trailer` - Agricultural trailers
      - `other` - Other agricultural implements
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Current status: `available`, `maintenance`, or `inactive`
    </ResponseField>
    
    <ResponseField name="registration_date" type="string">
      ISO 8601 timestamp of when the implement was registered
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata
  
  <Expandable title="Pagination Object">
    <ResponseField name="page" type="integer">
      Current page number
    </ResponseField>
    
    <ResponseField name="limit" type="integer">
      Records per page
    </ResponseField>
    
    <ResponseField name="total" type="integer">
      Total number of implements matching the query
    </ResponseField>
    
    <ResponseField name="totalPages" type="integer">
      Total number of pages
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:4000/api/implements?limit=10&page=1" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('http://localhost:4000/api/implements?limit=10&page=1', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
console.log(data.data); // Array of implements
```

```python Python
import requests

response = requests.get(
    'http://localhost:4000/api/implements',
    params={'limit': 10, 'page': 1}
)

implements = response.json()['data']
print(f"Found {len(implements)} implements")
```
</CodeGroup>

## Advanced Filtering

### Get Available Implements Only

To retrieve only implements with `status: available`, use the `/api/implements/available` endpoint:

```bash
curl -X GET "http://localhost:4000/api/implements/available?limit=10"
```

See the [Get Available Implements](/api-reference/implements/get#available-implements) section for details.

### Search with Filters

For advanced filtering by type, soil compatibility, and power requirements, use the `/api/implements/search` endpoint:

```bash
curl -X GET "http://localhost:4000/api/implements/search?type=plow&soilType=Loam&maxPower=60"
```

See [Search Implements](/api-reference/implements/get#search-implements) for more information.
